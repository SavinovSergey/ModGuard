# Структура проекта ModGuard

Краткое описание каталогов репозитория. Все пути указаны относительно корня проекта.

---

## Дерево каталогов

```
.
├── app/                    # Приложение: API, сервисы, модели, препроцессинг
│   ├── api/                # Эндпоинты FastAPI, схемы запросов/ответов
│   ├── core/               # Конфигурация, БД, очереди, кэш, model manager
│   ├── frontend/           # Демо веб-интерфейс (чат для проверки модерации)
│   ├── models/             # Модели классификации (базовые классы, toxicity, spam)
│   │   ├── toxicity/       # Модели токсичности: regex, tfidf, fasttext, rnn, bert
│   │   └── spam/           # Модели спама: regex, tfidf
│   ├── preprocessing/      # Предобработка текста (общая и для спама)
│   ├── features/           # Извлечение признаков (спам: капс, URL и т.д.)
│   ├── services/           # Бизнес-логика: классификация, сервисы токсичности/спама
│   ├── augmentation/       # Аугментация данных для обучения
│   ├── utils/
│   ├── loader.py           # Регистрация и загрузка моделей
│   └── main.py             # Точка входа FastAPI
├── scripts/                # Скрипты обучения, запуска сервисов, утилиты
│   ├── toxicity/           # Обучение и квантизация моделей токсичности
│   ├── spam/               # Обучение и валидация моделей спама
│   ├── run/                # Запуск воркера, listener Telegram, actions, healthcheck
│   ├── shared/             # Общие CLI, утилиты, загрузка данных (для скриптов)
│   └── augment_data.py     # Аугментация датасета
├── tests/                  # Тесты (API, модели токсичности/спама)
├── docs/                   # Документация (архитектура, дизайн, скрипты)
├── docker/                 # Конфигурация Docker и фронтенда
├── models/                 # Артефакты обученных моделей (не в git)
│   ├── toxicity/           # Подкаталоги: regex, tfidf, bert, rnn, fasttext, onnx...
│   └── spam/               # Подкаталоги: regex, tfidf
├── data/                   # Данные (датасеты, разбиения)
├── spam_data/              # Данные для обучения/валидации спама
├── requirements/           # Зависимости (разделены по профилям)
│   ├── core.txt            # API + worker (TF-IDF/regex)
│   ├── api.txt             # Только API (без ML)
│   ├── data.txt            # + pandas, datasets, валидация
│   ├── train.txt           # + optuna, обучение
│   ├── torch.txt           # + PyTorch-модели (RNN, BERT)
│   ├── bert.txt            # + BERT ONNX inference
│   ├── fasttext.txt        # + FastText
│   ├── gpu.txt             # + ONNX Runtime GPU
│   ├── test.txt            # pytest
│   └── all.txt             # Всё вместе
├── .env.example            # Пример переменных окружения
└── README.md
```

## Назначение ключевых каталогов

| Каталог | Назначение |
|---------|------------|
| **app/** | Основной код приложения: API, ядро (config, db, queue, cache, model_manager), модели токсичности и спама, препроцессинг, фичи, сервисы классификации, демо-фронтенд. |
| **app/api/** | Роуты FastAPI (`/api/v1/classify`, batch, health, stats), схемы запросов и ответов. |
| **app/core/** | Конфигурация (`config.py`), подключение к Postgres и Redis, очереди RabbitMQ, кэш модерации, менеджер моделей и fallback. |
| **app/models/** | Базовые классы моделей, реализации для токсичности (regex, TF-IDF, FastText, RNN, BERT) и для спама (regex, TF-IDF с фичами). |
| **app/preprocessing/** | Нормализация текста для токсичности и отдельный процессор для спама (сохранение регистра/ссылок где нужно). |
| **app/features/** | Ручные признаки для спама (капс, URL, email, TLD и т.д.) и правила CAPS_WORD. |
| **app/services/** | Сервисы классификации: объединение моделей токсичности и спама, запись в БД/кэш, публикация в очередь результатов. |
| **scripts/toxicity/** | Обучение моделей токсичности (BERT, TF-IDF, RNN, FastText), валидация, квантизация в ONNX. |
| **scripts/spam/** | Обучение и валидация моделей спама, оценка правил, разбиение и разметка данных. |
| **scripts/run/** | Запуск воркера классификации, Telegram listener, Telegram actions, healthcheck для RabbitMQ и воркера, валидация цепочки (validate_chain), очистка кэша Redis (clear_redis). |
| **scripts/shared/** | Общие аргументы CLI, функции (подбор порога, loss), загрузка и подготовка данных для скриптов. |
| **tests/** | Юнит- и интеграционные тесты API, моделей токсичности и спама. |
| **docker/** | Dockerfile и конфигурация для сборки и запуска сервисов. |
| **models/** | Директория для сохранения артефактов обученных моделей (веса, конфиги, векторайзеры); не коммитится в git. |

Скрипты запускаются из **корня проекта** (каталог, в котором находятся `app/`, `scripts/`, `models/`). Примеры команд см. в [SCRIPTS.md](SCRIPTS.md).
